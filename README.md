# hitalent_testovoe

REST API организационной структуры на Go.

Проект реализует сервис для хранения структуры компании: подразделений, вложенных
подразделений и сотрудников. Подразделения образуют дерево через `parent_id`, а
сотрудники привязаны к конкретному подразделению.

API позволяет:

- создавать подразделения верхнего уровня и дочерние подразделения;
- создавать сотрудников внутри подразделения;
- получать подразделение вместе с сотрудниками и поддеревом дочерних подразделений;
- менять название подразделения и переносить его в другое место дерева;
- удалять подразделение каскадно или с переносом сотрудников в другое подразделение.

В бизнес-логике учитываются ограничения из ТЗ: нельзя создать сотрудника в
несуществующем подразделении, нельзя сделать подразделение родителем самого себя,
нельзя создать цикл в дереве, а названия подразделений уникальны в пределах одного
родителя.

## Стек

- Go `net/http`
- PostgreSQL
- GORM
- goose migrations
- Docker Compose

## Архитектура

Проект разделен на слои:

- `cmd/api` — точка входа приложения;
- `internal/config` — чтение конфигурации из переменных окружения;
- `internal/db` — подключение к PostgreSQL через GORM;
- `internal/domain` — доменные модели;
- `internal/dto` — структуры запросов и ответов API;
- `internal/repository` — работа с базой данных;
- `internal/service` — бизнес-логика и валидация;
- `internal/transport/http` — HTTP-роутер, handlers и middleware;
- `migrations` — SQL-миграции goose.

## Запуск


docker compose up --build

API будет доступно на `http://localhost:8080`.

## Основные методы

- `POST /departments/` — создать подразделение
- `POST /departments/{id}/employees/` — создать сотрудника в подразделении
- `GET /departments/{id}?depth=1&include_employees=true` — получить подразделение, сотрудников и дерево дочерних подразделений
- `PATCH /departments/{id}` — изменить название или `parent_id`
- `DELETE /departments/{id}?mode=cascade` — удалить подразделение каскадно
- `DELETE /departments/{id}?mode=reassign&reassign_to_department_id=2` — удалить подразделение и перенести сотрудников в другое
